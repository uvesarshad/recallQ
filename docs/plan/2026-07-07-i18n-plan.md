# RecallQ i18n Plan

> Date: 2026-07-07
> Source: `docs/plan/2026-07-07-karakeep-audit-task-list.md`
> Scope: Internationalization plan for post-core stabilization.

## Direction
Keep v1 English-only until capture durability, import/export, mobile, and sync flows stabilize. Then add i18n in one deliberate pass instead of sprinkling string helpers through active feature work.

## Task List
1. Choose a message catalog library compatible with Next App Router and Expo.
2. Define route locale strategy for web; do not localize API paths.
3. Extract shared product terms for web, extension, mobile, CLI, and emails.
4. Add pseudo-locale tests for text overflow in dense feed/detail surfaces.
5. Add locale-aware date/time formatting for reminders, reader progress, and import/export timestamps.
6. Keep AI prompt locale explicit: user content can be any language, but enrichment output language should be a user preference.
7. Document translator workflow and block release on missing keys.

## First Locales
Start with `en`, then add `hi` only after mobile layout checks pass. Expand further based on market usage.
